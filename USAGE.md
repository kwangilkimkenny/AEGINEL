# AEGINEL Chrome Extension 사용 가이드

> **AEGINEL Guard** — AI 채팅 서비스에서 개인정보를 보호하고 위험한 프롬프트를 탐지하는 브라우저 확장 프로그램

---

## 1. 설치

### 1.1 빌드

```bash
cd apps/extension
pnpm install
pnpm build
```

빌드 완료 시 `dist/` 폴더가 생성됩니다.

### 1.2 Chrome에 로드

1. Chrome 주소창에 `chrome://extensions` 입력
2. 우측 상단 **"개발자 모드"** 토글 켜기
3. **"압축해제된 확장 프로그램을 로드합니다"** 클릭
4. `apps/extension/dist/` 폴더 선택
5. 툴바에 방패 아이콘이 나타나면 설치 완료

### 1.3 업데이트

코드 수정 후:

```bash
pnpm build
```

이후 `chrome://extensions` 페이지에서 AEGINEL Guard 카드의 **새로고침 버튼** 클릭.

## 3. 기본 기능 — 실시간 위험 탐지

### 3.1 동작 원리

1. 사용자가 LLM 채팅 입력창에 텍스트를 입력
2. 300ms 디바운스 후 9-Layer 탐지 엔진이 자동 스캔
3. 위험 감지 시 입력창 위에 **경고 배너** 표시
4. 차단 임계값(기본 60점) 초과 시 **전송 차단 + 모달** 표시

### 3.2 경고 배너

입력창 상단에 표시되며, 위험 수준별 색상이 다릅니다:

| 수준     | 색상 | 의미                   |
| -------- | ---- | ---------------------- |
| Low      | 초록 | 경미한 위험, 참고용    |
| Medium   | 노랑 | 주의 필요              |
| High     | 주황 | 높은 위험              |
| Critical | 빨강 | 심각한 위험, 전송 차단 |

배너에는 탐지된 카테고리, PII 개수, 위험도 점수가 표시됩니다. X 버튼으로 닫을 수 있습니다.

### 3.3 전송 차단

위험도 점수가 차단 임계값을 초과하면:

1. 전송 버튼 클릭 시 자동 차단
2. 모달 표시: 차단 사유, 탐지 카테고리
3. **"Go Back"** — 입력을 수정하러 돌아가기
4. **"Send Anyway"** — 위험을 인지하고 원본 그대로 전송

### 3.4 9-Layer 탐지 항목

| Layer | 탐지 대상                                         |
| ----- | ------------------------------------------------- |
| 1     | 유해 키워드 (한/영)                               |
| 2     | 탈옥 패턴 (DAN, 역할극, 개발자 모드)              |
| 3     | 프롬프트 인젝션 (시스템 프롬프트 오버라이드)      |
| 4     | 데이터 추출 시도 (API 키, 학습 데이터)            |
| 5     | 사회공학 (긴급 상황 사칭, 권위 사칭)              |
| 6     | 스크립트 우회 (초성/자모, 코드스위칭, 호모글리프) |
| 7     | 인코딩 공격 (Base64, Hex, ROT13, Leetspeak)       |
| 8     | 멀티턴 공격 (Crescendo 패턴, 교육 목적 프레이밍)  |
| 9     | 시맨틱 위험 (유해 행위 + 단계별 지시 결합)        |

## 4. PII Proxy (가명화) — 개인정보 보호

### 4.1 개요

LLM에 전송하는 텍스트에서 개인정보를 자동으로 감지하여 **형식 보존 가짜 값**으로 치환한 뒤 전송합니다. LLM 응답에 가짜 값이 포함되면 원본으로 자동 복원합니다.

**핵심 원칙:**

- 개인정보가 LLM 서버로 절대 전송되지 않음
- 매핑 테이블은 세션 메모리에만 존재 (영구 저장 안 함)
- 사용자 경험을 해치지 않음 (자동 보호, 비차단)

### 4.2 지원 PII 유형 (7종)

| PII 유형      | 예시 원본             | 가명 예시                         |
| ------------- | --------------------- | --------------------------------- |
| 주민등록번호  | `880101-1234567`      | `950315-2847391`                  |
| 신용카드번호  | `4111-1111-1111-1111` | `4829-3847-1926-5031` (Luhn 유효) |
| 이메일        | `user@company.com`    | `user_a7f2@example.com`           |
| 한국 전화번호 | `010-1234-5678`       | `010-8473-9152`                   |
| 국제 전화번호 | `+82-10-1234-5678`    | `+82-8473-9152`                   |
| US SSN        | `123-45-6789`         | `847-39-1524`                     |
| 여권번호      | `M12345678`           | `K84739152`                       |

### 4.3 두 가지 모드

#### Auto 모드 (기본)

1. 전송 버튼 클릭
2. PII 자동 감지 → 가명으로 치환
3. 가명화된 텍스트가 LLM에 전송됨
4. 파란색 보호 배너 표시: **"N개의 개인정보가 보호되었습니다"** (3초 후 자동 소멸)
5. LLM 응답에서 가명 → 원본 자동 복원

#### Confirm 모드

1. 전송 버튼 클릭
2. PII 감지 → **확인 모달** 표시
3. 모달에서 원본과 가명을 나란히 비교 확인:

```
[주민번호]  880101-1234567  →  950315-2847391
[이메일]    user@test.com   →  user_a7f2@example.com
```

4. **"보호 후 전송"** — 가명화된 텍스트로 전송
5. **"원본 전송"** — 원본 그대로 전송 (보호 안 함)

### 4.4 사용 예시

**입력:**

```
내 주민번호는 880101-1234567이고 이메일은 user@test.com입니다.
이 정보로 세금 신고 방법을 알려주세요.
```

**LLM에 실제 전송되는 텍스트 (auto 모드):**

```
내 주민번호는 950315-2847391이고 이메일은 user_a7f2@example.com입니다.
이 정보로 세금 신고 방법을 알려주세요.
```

**LLM 응답 (가명 포함):**

```
950315-2847391 주민번호로 홈택스에 접속하여...
user_a7f2@example.com으로 확인 이메일이 발송됩니다.
```

**사용자에게 표시되는 응답 (원본 복원):**

```
880101-1234567 주민번호로 홈택스에 접속하여...
user@test.com으로 확인 이메일이 발송됩니다.
```

## 5. 팝업 UI

툴바의 방패 아이콘을 클릭하면 팝업이 열립니다.

### 5.1 StatusCard (상단)

| 항목      | 설명                           |
| --------- | ------------------------------ |
| 보호 상태 | ON/OFF 토글 + 현재 사이트명    |
| Scans     | 총 스캔 횟수                   |
| Blocked   | 차단된 위협 수                 |
| Protected | PII Proxy로 보호된 개인정보 수 |

### 5.2 RiskMeter

마지막 스캔의 위험도를 반원 게이지(0-100)로 시각화합니다.

### 5.3 Recent Scans

최근 스캔 이력을 시간순으로 표시합니다. 각 항목에 위험도, 사이트, 카테고리가 표시됩니다.

## 6. 설정

팝업 하단의 **Settings** 섹션을 펼치면 아래 설정을 조절할 수 있습니다.

### 6.1 PII Detection

개인정보 탐지 기능 전체 ON/OFF.

### 6.2 PII Proxy (가명화)

| 설정        | 설명                                          | 기본값 |
| ----------- | --------------------------------------------- | ------ |
| 자동 가명화 | PII Proxy 기능 전체 ON/OFF                    | ON     |
| Mode        | `Auto` (자동 치환) / `Confirm` (확인 후 적용) | Auto   |
| 알림 배너   | 가명화 완료 시 파란색 배너 표시 여부          | ON     |

### 6.3 Block Threshold

위험도 점수 임계값 (20-100). 이 점수를 초과하면 전송이 차단됩니다.

- 기본값: 60
- 낮출수록 더 엄격하게 차단

### 6.4 Sensitivity

탐지 감도 배수 (0.5x-2.0x).

- 기본값: 1.0x
- 높일수록 더 민감하게 탐지

### 6.5 Detection Layers

9개 탐지 레이어를 개별적으로 ON/OFF할 수 있습니다.

### 6.6 Language

탐지 언어 설정: Auto (자동 감지) / English / Korean.

### 6.7 Clear Scan History

스캔 이력과 통계를 초기화합니다.

## 7. 자주 묻는 질문

### Q: PII Proxy로 보호된 정보가 어딘가에 저장되나요?

**아닙니다.** 매핑 테이블은 Service Worker의 메모리에만 존재합니다. 브라우저를 닫거나 확장 프로그램을 다시 로드하면 매핑이 초기화됩니다. 디스크, 서버, 클라우드 어디에도 저장되지 않습니다.

### Q: PII Proxy를 끄고 싶으면?

팝업 → Settings → PII Proxy (가명화) → **자동 가명화** 토글을 OFF로 전환하세요.

### Q: 가명화 없이 원본을 보내고 싶으면?

1. Mode를 `Confirm`으로 변경
2. 전송 시 확인 모달에서 **"원본 전송"** 선택

또는 PII Proxy 자체를 OFF로 설정하세요.

### Q: 특정 PII 유형만 보호하고 싶으면?

현재 PII Proxy는 PII Detection에서 활성화된 유형만 감지합니다. Settings → PII Detection 하위에서 개별 유형을 ON/OFF할 수 있습니다.

### Q: 경고 배너가 너무 자주 나타나요.

- Block Threshold를 높이면 차단 빈도가 줄어듭니다
- Sensitivity를 낮추면 탐지 민감도가 줄어듭니다
- 불필요한 Detection Layer를 OFF할 수 있습니다

### Q: 새로운 LLM 사이트를 추가하고 싶으면?

[개발자가이드.md](../../docs/핵심문서/개발자가이드.md)의 **11.3 사이트 어댑터 추가** 섹션을 참조하세요. `SiteAdapter` 인터페이스를 구현하고 `manifest.json`에 URL 패턴을 추가하면 됩니다.

## 8. 문제 해결

| 증상                  | 해결 방법                                                   |
| --------------------- | ----------------------------------------------------------- |
| 방패 아이콘이 안 보임 | 툴바 우측 퍼즐 아이콘 → AEGINEL Guard 핀 고정                 |
| 사이트에서 동작 안 함 | `chrome://extensions` → AEGINEL Guard 새로고침 버튼 클릭      |
| 빌드 에러             | `pnpm install` 재실행 후 `pnpm build`                       |
| 팝업이 빈 화면        | 개발자 모드에서 Service Worker "검사" 클릭 → 콘솔 에러 확인 |
| PII가 감지 안 됨      | Settings → PII Detection ON 확인, 해당 PII 유형 활성화 확인 |
